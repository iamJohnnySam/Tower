# Tower M3 Implementation Plan — Communication tab + gRPC Telegram bridge

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax. This plan spans TWO repos — Tower (`/home/atom/dev/Tower`) and MediaBox (`/home/atom/dev/MediaBox2026`). Each repo's tasks commit to that repo. The MediaBox changes are all behind an OFF-by-default flag, so MediaBox behavior is unchanged until the explicit cutover (Task 13), which is HELD for user go-ahead.

**Goal:** Move all Telegram I/O to Tower. Tower owns the bot token, the connection (long-poll), subscribers, and the message log, and exposes a gRPC bridge so MediaBox sends/receives Telegram via Tower. Full fidelity — text commands, notifications, AND interactive inline-button/callback flows. Add a Communication tab to Tower.

**Architecture:** Shared `.proto`. Tower is the gRPC **server** + the sole Telegram Bot API client. MediaBox dials in: outbound notifications are **unary RPCs**; inbound updates arrive via a **server-streaming RPC** (`StreamUpdates`). A `UseTowerTelegram` flag in MediaBox (off=today's local bot; on=gRPC client) and a `telegram.active` setting in Tower (gates Tower's long-poll loop) flip together at cutover — this avoids the Telegram "one getUpdates consumer per token" 409 conflict during the phased period.

**Tech Stack:** .NET 10, `Grpc.AspNetCore` (Tower server), `Grpc.Net.Client` + `Google.Protobuf` + `Grpc.Tools` (MediaBox client), Blazor Server, EF Core/SQLite. Port for gRPC: Tower already serves HTTP on 8888 (HTTP/1.1+2 cleartext (h2c)); gRPC will use a dedicated h2c endpoint on **5601** to keep concerns separate.

## The shared contract — `protos/tower_telegram.proto`

Lives in Tower at `protos/tower_telegram.proto` and is COPIED verbatim into MediaBox at `MediaBox2026/Protos/tower_telegram.proto` (Task 8). Keep them byte-identical.

```proto
syntax = "proto3";
option csharp_namespace = "Tower.Telegram.Grpc";
package tower.telegram;

service TowerTelegram {
  rpc StreamUpdates(StreamRequest) returns (stream Update);      // Tower -> MediaBox (incoming msgs + callbacks)
  rpc SyncSubscribers(SubscriberList) returns (Ack);             // MediaBox -> Tower one-time subscriber import
  rpc SendMessage(SendMessageRequest) returns (SendResult);
  rpc SendPhoto(SendPhotoRequest) returns (SendResult);
  rpc SendInlineKeyboard(InlineKeyboardRequest) returns (SendResult); // SendResult.message_id set
  rpc EditMessage(EditMessageRequest) returns (SendResult);
  rpc AnswerCallback(AnswerCallbackRequest) returns (Ack);
  rpc ListSubscribers(Empty) returns (SubscriberList);
}

enum Audience { CHAT = 0; ADMIN = 1; SUBSCRIBERS = 2; }

message StreamRequest { string client_id = 1; }
message Update {
  oneof kind {
    IncomingMessage message = 1;
    CallbackQuery   callback = 2;
  }
}
message IncomingMessage { int64 chat_id = 1; string text = 2; string username = 3; string first_name = 4; string last_name = 5; }
message CallbackQuery   { int64 chat_id = 1; string callback_id = 2; string data = 3; int32 message_id = 4; }

message SendMessageRequest    { Audience audience = 1; int64 chat_id = 2; string text = 3; string parse_mode = 4; }
message SendPhotoRequest      { Audience audience = 1; int64 chat_id = 2; string photo_url = 3; string caption = 4; repeated ButtonRow buttons = 5; string parse_mode = 6; }
message InlineKeyboardRequest { int64 chat_id = 1; string text = 2; repeated ButtonRow buttons = 3; string parse_mode = 4; }
message EditMessageRequest    { int64 chat_id = 1; int32 message_id = 2; string text = 3; string parse_mode = 4; repeated ButtonRow buttons = 5; }
message AnswerCallbackRequest { string callback_id = 1; string text = 2; }

message ButtonRow { repeated Button buttons = 1; }
message Button    { string text = 1; string callback_data = 2; }

message SendResult     { bool ok = 1; int32 message_id = 2; string error = 3; }
message SubscriberList { repeated Subscriber subscribers = 1; }
message Subscriber     { int64 chat_id = 1; string name = 2; bool is_admin = 3; }
message Ack   { bool ok = 1; }
message Empty {}
```

Note on `EditMessage`/`SendInlineKeyboard` chat_id: MediaBox's current `ITelegramNotifier.EditMessageAsync(messageId, text)` and `SendInlineKeyboardAsync(text, buttons)` don't pass a chat id — the local service tracks a default target chat. The MediaBox gRPC notifier (Task 9) MUST resolve the target chat (admin chat id, retrieved once from Tower) and pass it. Read the existing `TelegramBotService` methods to replicate the exact target/parse-mode behavior.

### Task 4: TelegramHub — coordinator (poll loop + fan-out + message log)

**Files:** `src/Tower.Core/Telegram/TelegramHub.cs` (singleton), `src/Tower.Core/Workers/TelegramPollWorker.cs`, extend `LiveState` (Telegram comms snapshot: bot active?, last error, recent message log, connected gRPC clients count).

`TelegramHub` is the in-memory center:
- Holds the set of connected gRPC `StreamUpdates` subscribers (each an `IServerStreamWriter<Update>` + a channel) — `RegisterClient`/`UnregisterClient`, and `BroadcastUpdate(Update)`.
- `TelegramPollWorker` (BackgroundService) runs the long-poll loop ONLY when `telegram.active`=="true" (read via scoped SettingsService) AND `telegram.bot_token` set; otherwise idles (checks every 5s). When active: `TelegramApi.GetUpdatesAsync`, parse, and for each update: if it's a `/start` from an unknown chat → auth flow via SubscriberService (mirror MediaBox's current new-subscriber handling — read `TelegramBotService.HandleMessageAsync`); persist to the `TelegramMessage` log (direction=in); then `hub.BroadcastUpdate(...)` to forward to MediaBox. Advance offset.
- Outbound routing: `SendAsync(audience, chatId, text, ...)` resolves audience (CHAT→chatId, ADMIN→SubscriberService.GetAdmin, SUBSCRIBERS→fan-out over ListActive) and calls `TelegramApi`; persists to log (direction=out); returns SendResult. Same for photo/keyboard/edit/answer.
- Push a comms snapshot into LiveState for the Communication tab.

- [ ] Implement (no full unit test — integration-verified in Task 13; a small test for audience resolution is encouraged); build; commit: `"Add TelegramHub coordinator + gated poll worker + message log"`
